# Pacific Vector — Phase 2 Design Direction

Research notes on how to make Phase 2 (favicon, OG tags, About/Contact pages, footer,
mobile) feel professional, trustworthy, modern and distinctive — without sliding into
the generic "AI template" look. Goal: by the time Phase 2 is done, every page should
feel like it belongs to the same considered, analyst-grade product.

---

## 1. The good news: the current design is already on the right track

`index.html` already avoids the most common AI-template tells:

- **Monospace type (IBM Plex Mono)** instead of the default Inter/system-sans —
  reads as "terminal / dispatch / analyst desk," which fits a geopolitical
  intelligence product perfectly.
- **No gradients, no rounded cards, no drop shadows, no glassmorphism.**
- **Restrained palette** — navy (`#1a3a5c`), cream/off-white background (`#f8f8f6`),
  near-black ink, grey rules. No purple, no neon accent.
- **Hairline rules and uppercase letter-spaced labels** ("— JAPAN–CHINA",
  "// SO WHAT") — this reads like a classified cable / Bloomberg Terminal /
  field report, which is a strong, distinctive identity.

The job for Phase 2 is to **extend this identity consistently** to every new
surface (favicon, social previews, About, Contact, footer) — not introduce a
different "marketing site" style on top of it. The biggest risk right now isn't
that the site looks bad — it's that new pages get built in a generic
"SaaS landing page" style that clashes with the dispatch aesthetic already
established.

---

## 2. What to actively avoid (the "generic AI site" tells)

Based on current research into what makes 2026-era AI-generated sites look
samey, watch out for:

- **Purple/blue gradient backgrounds or buttons** (the default LLM palette).
  Pacific Vector's navy/cream is already a good antidote — keep it.
- **Inter font for everything**, especially pairing a generic sans headline
  with a generic sans body. The mono font is already a differentiator —
  consider a **second, contrasting typeface for headlines/display use**
  (see typography section) rather than defaulting to system sans for new
  pages like About/Contact.
- **Rounded cards with soft shadows ("8px/16px border-radius + shadow" pattern)**
  — the current site correctly uses hairline borders and flat fills. Continue
  this on About/Contact (e.g. profile-card style blocks, not "feature cards").
- **Colored left-border "callout" blocks in purple/blue** — Pacific Vector
  already has this pattern (`.so-what` with navy left border) but it's used
  sparingly and meaningfully (signals "analyst judgment"), not decoratively.
  Don't add more of these just for visual interest.
- **Three-icon feature grids** — common on About pages generated by AI
  ("Why choose us" with 3 icons). Avoid. If About needs to communicate
  pillars (Monitor/Ask/Brief), present them as a short numbered/lettered
  list or table row format consistent with the brief's own section styling,
  not icon cards.
- **Stock photography of people in suits shaking hands / globes / generic
  "geopolitics" stock imagery.** If imagery is wanted at all, consider:
  abstract/maps, redacted-document textures, or simply none — text-led design
  is itself a 2026 trend and suits this product's "serious analyst" tone.

---

## 3. Typography direction

Current: IBM Plex Mono throughout (300 weight body, 500 weight emphasis).

Recommendation for Phase 2 expansion:
- **Keep IBM Plex Mono for UI chrome, labels, metadata, nav** — it's the
  signature element (like a dateline/timestamp font).
- For **About/Contact page body copy**, consider introducing **IBM Plex Serif
  or a similar serif** (same type family, so it pairs naturally) for longer-form
  reading — serif body text reads as "considered editorial" (Economist,
  Foreign Affairs) vs. "SaaS product." Mono stays for headers/labels/dates,
  serif for paragraphs of prose. This single pairing decision would make About
  feel like an essay/dossier rather than a landing page, while staying in the
  same type family as the rest of the site.
- Avoid introducing a third typeface. Two is enough: one mono (interface/labels),
  one serif (long-form prose), both from a family that shares IBM Plex's
  design language for cohesion.

---

## 4. Color & texture direction

- Keep the existing palette as the base (navy / cream / ink / grey rule).
- For **OG image / social preview card**: dark navy background with cream
  monospace text mimicking a "brief header" — e.g. the masthead treatment
  (PACIFIC[VECTOR] wordmark + dateline + opening line excerpt) rendered as an
  image. This reuses the existing visual language rather than inventing a new
  "marketing" look, and looks distinctive in a LinkedIn/X feed full of generic
  blue gradient cards.
- For **favicon**: a simple monochrome mark — e.g. a stylised compass
  bearing/vector arrow, or just "PV" in the IBM Plex Mono, set in navy on
  cream (or inverted for dark mode browser tabs). Keep it geometric and
  flat — no gradients or 3D bevels.
- Optional subtle texture: a very faint dot-grid or coordinate-grid background
  (referencing maps/charts) could reinforce the "intelligence/cartography"
  feel without being loud — used sparingly (e.g. About page hero band only),
  consistent with the "tactile, engineered, human-made" trend research
  surfaced (hairline grids, 1px borders, raw geometry) rather than soft
  blurred shapes.

## 5. Page-by-page notes for Phase 2 items

**Favicon**
- Geometric mark in navy, works at 16px. "PV" monogram or a minimal compass/
  vector-arrow icon. Provide light/dark variants if feasible.

**OG / social metadata**
- Static OG image (1200x630) styled as a "brief masthead" card — navy bg,
  cream mono type, logo + tagline + today's date format. Avoid stock photos.

**About page**
- Lead with the mission/insight in the same "lede" style as the brief
  (`.lede` block with navy left border).
- Use serif body copy for the founder/background narrative (per typography
  note). Given the founder's anonymity constraint, frame this around the
  *product's* methodology and editorial standards rather than a personal bio
  — "Who's behind Pacific Vector" can describe the analytical approach,
  sourcing standards, and the editorial voice rather than naming an individual.
- Present the three pillars (Monitor/Ask/Brief) as a simple labeled list/table
  in the existing pillar-label style — not icon cards.

**Contact page**
- Keep it minimal: a short line of text + mailto link or simple form, styled
  with the same input/button treatment as the existing subscribe form
  (dark navy band, mono labels). Avoid adding a generic "contact card" with
  icons for phone/email/address — this isn't that kind of business yet.

**Footer**
- Extend the existing footer style (small mono caps, grey, justified ends)
  with links to About / Archive / Contact / Subscribe — same treatment as
  nav links, not a new "mega footer" with columns.

**Mobile review**
- The existing CSS already has a `@media (max-width: 600px)` block — extend
  this pattern for new pages rather than introducing a separate mobile
  framework. Check that the subscribe banner, nav, and any new About/Contact
  layouts collapse the same way (stacked, full-width).

---

## 6. One-line design principle to keep everything consistent

> **"Field report, not landing page."** Every new page should look like
> another page out of the same analyst's notebook — same type, same hairline
> rules, same restrained navy/cream palette, same uppercase mono labels —
> rather than a separate marketing site bolted onto the brief.
